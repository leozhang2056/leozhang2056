## Directory Structure Standard

Each project should follow this structure:

```
projects/
└── [project-name]/
    ├── README.md          # Project documentation (English only)
    ├── facts.yaml         # Structured project metadata
    ├── images/            # Screenshots and diagrams
    │   ├── *.png
    │   ├── *.jpg
    │   └── README_PLACEHOLDER.md  # If no images yet
    └── [source code]      # Optional: actual code
```

---

## Naming Conventions

### Image Files
- Use lowercase with hyphens: `mobile-app-home-screen.png`
- Avoid spaces and special characters
- Descriptive names: `dashboard-overview.png` not `img1.png`

### README Sections
- Overview
- Key Features
- Architecture
- Technologies
- Key Achievements
- Responsibilities
- Challenges & Solutions
- Results & Impact
- Evidence (screenshots)
- Skills Demonstrated
